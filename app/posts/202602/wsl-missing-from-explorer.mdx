ある日突然、Windows のエクスプローラーのサイドバーから Linux の項目が消えていました  
WSL 上のファイルにアクセスしようとすると「要素が見つかりません。」というエラーが出る始末  

今回はこの問題を `wsl --update` で解決できたので備忘録として残します  

## ■ 症状

エクスプローラーのサイドバーを見ると、通常あるはずの「Linux」の項目が表示されていません  

![エクスプローラーに Linux が表示されていない](/images/posts/202602/wsl-missing-from-explorer.png)

さらに、WSL のフォルダ（Ubuntu など）にアクセスしようとすると Explorer.EXE から「要素が見つかりません。」というエラーダイアログが表示されます  

![要素が見つかりませんエラー](/images/posts/202602/wsl-element-not-found.png)

WSL 自体はターミナルから `wsl` コマンドで起動できるので、ディストリビューションが壊れたわけではなさそうです  
エクスプローラーとの統合部分に問題がある様子  

## ■ 解決方法

PowerShell または コマンドプロンプトを管理者権限で開いて、以下を実行します  

```bash title="WSL をアップデート" showLineNumbers
wsl --update
```

アップデート完了後にエクスプローラーを開き直すと、サイドバーに Linux が復活し、WSL 上のファイルに正常にアクセスできるようになりました  

## ■ なぜ起きたのか

おそらく Windows Update や WSL のバージョン不整合が原因です  
WSL はカーネルやシェル統合コンポーネントが Windows 本体とは別にアップデートされるため、Windows 側のアップデート後に WSL 側が追従できていないとエクスプローラー統合が壊れることがあるようです  

`wsl --update` を実行すると、WSL カーネルとその関連コンポーネント（エクスプローラー統合を含む）が最新版に更新されるので、これで不整合が解消されます  

## ■ まとめ

エクスプローラーから Linux が消えたら、まず `wsl --update` を試してみてください  
自分の場合はこれだけで即座に解決しました  
それでもダメな場合は `wsl --shutdown` で WSL を再起動してからもう一度試すと良いかもしれません
